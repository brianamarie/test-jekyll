• before_deploy
    before_deploy TL;DR

    You can run steps before a deploy by using the before_deploy phase. A non-zero exit code in this command will mark the build as errored. Optional.

    What is before_deploy?

    The before_deploy can be used to prepare your deployment, if need be. If you want to make sure that this only runs if your build was successful, you can check the $TRAVIS_TEST_RESULT environment variable.

    When is before_deploy necessary?

    Use before_deploy if there is specific steps that need to be completed before it's okay to deploy, but those steps don't belong in the earlier script portions.

    before_deploy examples
    
      before_deploy:
        - test $TRAVIS_TEST_RESULT = 0 \# check that build passed successfully
          && gulp build \
          && cd dist \
          && npm install --prod \
          && cd $TRAVIS_BUILD_DIR \
          && tar -zcvf amber-storm-$TRAVIS_BRANCH.tar.gz dist
