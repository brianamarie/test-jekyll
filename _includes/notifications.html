    notifications TL;DR

    Manage notification preferences. Optional.

    What are notifications?

    By default, Travis sends an email notification whenever a status changes. You can change that within the config file.
    For more information on notifications, see the detailed documentation.

    When are notifications necessary?

    The notifications section is optional. You'd add this section if you want to change the notifications generated by Travis when it completes the builds, tests, and deployments launched from this travis.yml file.

    notifications example
    
        notifications:
          email:
            recipients:
              - one@example.com
              - other@example.com
            on_success: never # default: change
            on_failure: always # default: always
